## 2. 데이터 현황 활용

### 2.1 기존 각도별 scene 구조

각도별 scene 폴더명에서 **포신 각도 라벨**을 추출할 수 있습니다.

| scene 예시 | 추정 앙각 |
|------------|-----------|
| `90식_45도각도_포신` | 45° |
| `90식_90도각도_포신` | 90° |
| `90식_정면각도-포신` | 0° (정면) |
| `K1A1_45도각도` | 45° |
| `tiger/1촬영30도` | 30° |
| `tiger/1촬영60도` | 60° |
| `tiger/1촬영90도` | 90° |

> **주의**: "90도"는 포신 앙각이 아니라 **촬영 각도**(측면 뷰)일 수 있습니다. 실제 앙각은 수동 검증 또는 3D GS 렌더와 대조해 확인 필요.

### 2.2 앙각 라벨 데이터셋 구성


## 3. 접근 A: 이미지 → 각도 회귀 (권장 먼저)

### 3.1 모델 구조

- **백본**: ResNet18/34 또는 EfficientNet-B0 (경량)
- **헤드**: FC → 1 dim (앙각만) 또는 2 dim (앙각 + 방위각)
- **손실**: Smooth L1 (L1Loss) 또는 MSE
- **메트릭**: MAE(°), ±5° 이내 정확도

### 3.2 학습 파이프라인 (PyTorch)

```python
# 예시: cannon_angle_regression/train.py
import torch
import torch.nn as nn
from torchvision import models

class CannonAngleRegressor(nn.Module):
    def __init__(self, backbone="resnet18", pretrained=True):
        super().__init__()
        base = models.resnet18(weights="IMAGENET1K_V1" if pretrained else None)
        self.features = nn.Sequential(*list(base.children())[:-1])
        self.fc = nn.Linear(512, 1)  # elevation_deg

    def forward(self, x):
        x = self.features(x)
        x = x.flatten(1)
        return self.fc(x).squeeze(-1)
```

### 3.3 데이터 증강

- **회전**: 제한적 (전차 방향 유지) — ±15° 수평 회전만
- **색상**: 밝기/대비, 노이즈
- **크롭**: 랜덤 크롭 후 resize (전차 중심 유지)

### 3.4 추론 API (ai-inference 확장)

```python
# POST /cannon/angle/predict
# 입력: 이미지 파일 (또는 base64)
# 출력: { "elevation_deg": 12.3, "confidence": 0.85, "method": "image" }
```

## 4. 접근 B: 3D 대조 (3D GS 기반)

### 4.1 전제 조건

- **레퍼런스 3D GS**: 0° 앙각 전차 `point_cloud.ply` (또는 정면각도 scene 학습 결과)
- **포신 축 정의**: Phase 0.4 — 레퍼런스에서 `ref_barrel_direction` 추출

### 4.2 절차

1. **관측 3D GS 생성**: 새 이미지(들)로 COLMAP → 3D GS 학습 (또는 단일/소수 뷰 보정)
2. **정합(Registration)**: 레퍼런스와 관측 3D GS를 ICP 등으로 정렬
3. **포신 축 추출**: 관측 3D GS에서 포신 영역 가우시안 식별 → 주축/직선 피팅
4. **각도 계산**: `ref_barrel_direction`과 관측 포신 방향의 회전 각도 → 앙각(°)

### 4.3 포신 영역 식별

- **위치 기반**: 포신은 전차 상부 전방에 위치 — 바운딩 박스 또는 휴리스틱
- **형상 기반**: 가우시안 공분산의 주축이 길쭉한 것(원통형) 필터링
- **수동 마스크**: 초기에는 포신 영역을 수동으로 마스킹해 가우시안 서브셋 정의

### 4.4 기술 스택

| 도구 | 용도 |
|------|------|
| Open3D | PLY 로드, ICP, 포인트클라우드 처리 |
| numpy | 벡터 연산, 각도 계산 |
| (선택) gaussian-splatting | 3D GS 로드 및 렌더 |
