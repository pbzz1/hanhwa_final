## 1. ZIP 구조 (전차데이터.zip)

```
전차데이터/
├── 1. 이미지 촬영/          # 영상 파일
├── 2. 크롭/                 # 크롭된 이미지
├── 3. 라벨링/               # 3D GS용 이미지 (이 폴더 사용)
│   ├── 90식/
│   │   └── 라벨링/
│   │       ├── 90식_45도각도_포신/
│   │       │   ├── 90식_45도각도_포신0도/   *.jpg
│   │       │   ├── 90식_45도각도_포신45도/  *.jpg
│   │       │   ├── 90식_45도각도_포신90도/  *.jpg
│   │       │   ├── 90식_45도각도_포신135도/ *.jpg
│   │       │   └── 90식_45도각도_포신180도/ *.jpg
│   │       ├── 90식_90도각도_포신/  (동일 구조)
│   │       ├── 90식_정면각도-포신/  (동일 구조)
│   │       └── 정면각도/             (이미지 없음)
│   ├── K1A1/라벨링/...

│   ├── M1A2/라벨링/...

│   └── tiger/라벨링/
│       ├── 1촬영30도/
│       │   └── Tiger_촬영0도/  *.jpg
│       ├── 1촬영60도/
│       └── 1촬영90도/
└── dataset.yaml
```

## 2. 압축 해제 후 경로 (Colab)

`unzip 전차데이터.zip -d /content/data/tanks` 실행 시:

| 경로 | 설명 |
|------|------|
| `/content/data/tanks/전차데이터/3. 라벨링` | **라벨링 루트** (이 경로를 src로 사용) |

## 5. 이미지 중첩 구조

각 pose 폴더 아래에 **촬영 각도별 서브폴더**가 있음:
- `90식_45도각도_포신/90식_45도각도_포신0도/*.jpg`
- `90식_45도각도_포신/90식_45도각도_포신45도/*.jpg`
- ...

파일명이 서브폴더마다 겹칠 수 있으므로, 복사 시 **고유 이름** 생성 필요.
