# 포신 각도 학습 및 지도 포격/위험 지역 예측 — 구현 커리큘럼

## 목표 요약

1. **3D Gaussian Splatting으로 전차 3D 모델 학습**: 다중 시점 이미지 → SfM + 3D GS 최적화 → 전차 레디언스 필드(실시간 렌더 가능)
2. **포신 각도 학습**: 학습된 3D GS(또는 렌더 뷰)와 관측을 대조해 **현재 포신 각도(°)** 예측
3. **포신 각도 검증**: 예측 각도의 신뢰도 및 검증
4. **지도 연동**: 포격 위치 예측 및 위험 지역 표시

---

## ⭐ 3D Gaussian Splatting 도입 (기존 방식 대체)

**제거할 기존 방식**: MASt3R 기반 멀티뷰 복원, ORB + triangulation 기반 2-view 복원 (`reconstruct-3d`, `reconstruct-3d-multi`).

**기존 코드 변경 (ai-inference)**  
- `main.py`에서 MASt3R 로딩/상태(`_load_mast3r_runtime`, `MAST3R_STATE`), `reconstruct-3d`, `reconstruct-3d-multi` 엔드포인트 및 관련 헬퍼 제거 또는 비활성화.  
- 3D 복원은 **별도 3D GS 파이프라인**(COLMAP → 3D GS 학습 스크립트)으로 수행하고, 학습된 3D GS 결과(가우시안 파라미터 또는 export된 포인트/메쉬)를 API에서 로드해 포신 축 추출·각도 대조에 사용하도록 설계.

**3D Gaussian Splatting 요약** [Kerbl et al., SIGGRAPH 2023]:

- **입력**: 정적 장면의 다중 시점 이미지 + **SfM** 으로 얻은 카메라 캘리브레이션 및 **스파스 포인트클라우드**
- **표현**: 장면을 **3D 가우시안** 집합으로 표현 (위치 μ, 이방성 공분산 Σ, 불투명도 α, 구면조화 SH로 뷰 의존 색상)
- **최적화**: 렌더링 vs 학습 뷰 비교 → L1 + D-SSIM, **적응 밀도 제어**(clone/split)로 품질 유지
- **렌더링**: 2D 투영 후 타일 기반 α-blending → **실시간 1080p** 가능
- **전차 적용**: 전차만 담은 이미지로 3D GS 학습 → 포신 영역 가우시안에서 **포신 축** 추출 → 레퍼런스 0° 정의 및 각도 대조에 활용

## Phase 0: 사전 준비 (1~2주)

### 0.1 환경 및 용어 정리

- **포신 각도**: 주로 **앙각(elevation)**, 필요 시 **방위각(azimuth)** 도 정의
- **좌표계**: 카메라/이미지 좌표 → 3D 월드(또는 전차 로컬) → 지도(위/경도)
- **레퍼런스**: 3D GS로 학습한 0° 전차 모델 및 그로부터 추출한 **포신 방향 벡터**

### 0.2 데이터 확보

| 항목 | 설명 |
|------|------|
| 전차 다중 시점 이미지 | 동일 전차를 여러 각도에서 촬영한 이미지(최소 20~50장). 0° 앙각 상태 세트를 레퍼런스 3D GS 학습에 사용 |
| (선택) 다양한 앙각 데이터 | 0°, 5°, 10°, 15° 등 포신 앙각별 이미지 → 각도 라벨과 함께 보관 (이미지 회귀용) |
| (선택) 지도/위치 | 전차 위치(위·경도 또는 UTM), 지형 데이터(고도 등). 포격 거리 추정에 사용 |

### 0.3 3D Gaussian Splatting 파이프라인 구축

- **SfM**: COLMAP으로 이미지에서 카메라 파라미터 + 스파스 포인트클라우드 생성
- **3D GS 학습**: [gaussian-splatting](https://github.com/graphdeco-inria/gaussian-splatting) 등으로 전차 장면 학습 (입력: COLMAP 결과, 출력: 3D 가우시안 파라미터)
- **레퍼런스 "0°" 전차 3D GS**: 포신 0° 고정 상태 이미지들만으로 3D GS 학습 → 이 모델을 레퍼런스로 사용

### 0.4 레퍼런스 3D GS에서 “포신 축” 정의

- 학습된 3D GS에서 **포신 영역 가우시안** 식별(위치/형상 기반 또는 수동 마스크)
- 해당 가우시안들의 **주축** 또는 직선 피팅으로 **포신 중심축 방향** 계산 → `ref_barrel_direction` (0° 기준)
- 필요 시 포신 회전축(요축) 정의 → 방위각 계산에 사용

## Phase 1: 포신 각도 추정 모델 (2~3주)

### 1.1 접근 A: 이미지 → 각도 회귀 (권장 먼저)

- **입력**: 전차가 크롭된 단일 이미지 (또는 YOLO bbox 크롭)
- **출력**: 앙각 1개(°), 필요 시 방위각 1개(°)
- **데이터**: (이미지, 실제_앙각) 쌍. 실제 앙각은 시뮬레이션/3D 렌더링 또는 수동 라벨링
- **모델**: ResNet/EfficientNet 등 백본 + 회귀 헤드(1 or 2 dim), 또는 비디오 프레임 연속성 활용 시 LSTM/Transformer

**체크리스트**

- [ ] 앙각 라벨이 붙은 데이터셋 구성 (최소 100~500장, 각도 스팬 넓을수록 유리)
- [ ] train/val split, 데이터 증강(회전 제한 또는 각도 보정)
- [ ] MSE 또는 Smooth L1 loss로 학습, 메트릭: MAE(°), ±5° 이내 정확도
- [ ] 추론 API: 이미지 업로드 → `elevation_deg`, `azimuth_deg` (선택) 반환

### 1.2 접근 B: 3D 대조 (3D Gaussian Splatting 기반)

- **입력**: 관측 장면의 **3D GS** (새 이미지들로 학습한 3D 가우시안) 또는 3D GS 렌더 뷰
- **절차**:
  1. **레퍼런스 3D GS**(0° 전차)와 관측 3D GS를 **정렬**(가우시안 위치/공분산 기반 ICP 또는 feature 정합)
  2. 관측 3D GS에서 **포신 영역 가우시안** 식별 후 **포신 축 추출**(주축 또는 직선 피팅)
  3. 레퍼런스 포신 방향(`ref_barrel_direction`)과 비교해 **회전 각도** 계산 → 앙각(°)
- **장점**: 3D GS는 명시적 가우시안이라 포신 형상/방향 추출에 유리. 실시간 렌더로 다양한 뷰 생성 가능.

**체크리스트**

- [ ] 레퍼런스 3D GS 로드 및 포신 축 정의(Phase 0.4)
- [ ] 관측 3D GS 학습 파이프라인(또는 단일/소수 뷰 보정) 및 포신 영역 필터링
- [ ] 정합(registration) 파이프라인 및 앙각 계산 공식화
- [ ] (선택) 이미지 회귀(1.1)와 3D GS 기반 각도 앙상블

### 1.3 포신 영역만 쓰는 하이브리드

- 2D에서 포신 영역 세그멘테이션(세그멘테이션 모델 또는 3D 투영 마스크) → 포신만 크롭
- 크롭 이미지를 1.1처럼 회귀 모델에 넣어 각도 예측
- 또는 포신 크롭으로 3D 포인트만 추출해 1.2처럼 축 추출

## Phase 2: 포신 각도 검증 (1주)

### 2.1 신뢰도 및 일관성

- **이미지 회귀**: 예측에 대한 불확실성(베이즈적 접근 또는 MC Dropout 등)
- **3D 대조**: 정합 오차(ICP residual), 포신 축 피팅 오차
- **규칙 기반**: 물리적 범위(예: -5° ~ +30°) 이탈 시 “검증 실패” 플래그

### 2.2 다중 시점/다중 프레임 검증

- 여러 시점에서 각도 예측 후 분산/이상치 제거(중앙값 등)
- 비디오면 시간축 스무딩 또는 간단한 필터(칼만 등)

### 2.3 API 및 UI

- `GET/POST /cannon/angle/verify` 같은 엔드포인트에서 `angle_deg`, `confidence`, `method`(image | 3d | hybrid) 반환
- 프론트에서 “포신 각도: 12° (검증됨)” 형태로 표시

## Phase 3: 지도 포격 위치 예측 및 위험 지역 (2~3주)

### 3.1 전제 조건

- **전차 위치**: 위·경도(또는 UTM) — 수동 입력, 메타데이터, 또는 (미래) 지도 매칭
- **포신 각도**: Phase 1/2에서 얻은 앙각(°), 필요 시 방위각(°)
- **포격 거리**: 탄도 모델(초기속도, 각도, 공기저항) 또는 **단순화**: 각도–거리 룩업 테이블/경험식

### 3.2 탄도/사거리 모델 (단순 버전)

- 각도 θ에 따른 사거리 \( d(\theta) \) 를 고정 초기속도·중력 가정으로 계산
  - 예: \( d \propto v_0^2 \sin(2\theta) / g \) (저항 무시)
- 또는 실측/시뮬레이션 테이블: `(angle_deg -> range_m)` 매핑

### 3.3 지도 위 포격점 좌표

- 전차 위치 \((lat_0, lon_0)\), 방위각 \(\phi\), 사거리 \(d\)
- 목표점:  
  - 위도/경도 보정 공식(예: Haversine 또는 평면 근사)으로  
  - `(lat_1, lon_1)` 계산
- 고도 있으면 지형 보정(선택): 지형 데이터로 고도 보정 후 거리/방향 재계산

### 3.4 위험 지역 표시

- **포격점**: 단일 점 마커
- **위험 지역**: 포격점 중심 반경 \(R\) (예: 50 m, 100 m) 원/폴리곤
- 불확실성 반영: 각도/위치 오차를 반영해 타원 또는 여러 반경으로 표시

### 3.5 지도 스택 선택

| 옵션 | 용도 |
|------|------|
| Leaflet + OpenStreetMap | 2D 웹 지도, 마커/원/폴리곤 |
| Mapbox GL JS | 벡터 타일, 커스텀 스타일 |
| Google Maps API | 위성/도로 뷰, API 할당량 주의 |
| Cesium (선택) | 3D 지구, 고도·지형 강조 |

### 3.6 백엔드 API

- `POST /cannon/fire-prediction`
  - 입력: `tank_lat, tank_lon, elevation_deg, azimuth_deg` (및 선택: 초기속도, 탄종)
  - 출력: `impact_lat, impact_lon, range_m, danger_zone_radius_m` (및 GeoJSON 폴리곤)

### 3.7 프론트엔드

- 지도 컴포넌트에서 전차 마커, 예측 포격점 마커, 위험 지역(원/폴리곤) 렌더링
- 포신 각도 슬라이더/입력으로 실시간 업데이트(선택)

## Phase 4: 통합 및 운영 (1~2주)

### 4.1 파이프라인 통합

- **이미지/영상 업로드** → YOLO 전차 검출 → (선택) 3D 복원  
  → **포신 각도 추정** → **검증** → **전차 위치 + 각도**로 **포격/위험 지역** 계산  
  → **지도에 표시**

### 4.2 설정 및 보안

- 전차 기본 위치, 탄종별 사거리 테이블, 위험 반경 기본값을 설정 파일/환경변수로 분리
- API 인증·속도 제한 검토

### 4.3 문서화 및 테스트

- 각 Phase별 단위 테스트(각도 예측, 탄도 계산, 좌표 변환)
- API 스펙(OpenAPI) 및 사용 예시 문서

```
Phase 0 (준비)
    → COLMAP + 3D GS로 전차 레디언스 필드 학습, 레퍼런스 "0°" 3D GS·포신 축 정의, (선택) 앙각 라벨 데이터 수집

Phase 1 (각도 추정)
    → 1.1 이미지 회귀 먼저 구현 (빠른 검증). 3D GS로 렌더한 뷰로 라벨/증강 가능
    → 1.2 3D GS 기반 3D 대조(레퍼런스 vs 관측)로 품질 개선/검증

    → 신뢰도·다중 시점 검증, API 정리

Phase 3 (지도)
    → 탄도/사거리 → 포격점 좌표 → 지도 위험 지역 표시

Phase 4 (통합)
    → 전 단계 연결, 설정, 테스트, 문서화
```

## 기술 스택 제안

| 영역 | 추천 |
|------|------|
| 각도 회귀 | PyTorch, torchvision (또는 기존 YOLO와 같은 repo 내 학습) |
| 3D 장면 표현 | **3D Gaussian Splatting** (공식 구현 또는 diff-gaussian-rasterization), COLMAP(SfM) |

| 지도 | Leaflet + React (또는 기존 프론트 스택에 맞춤) |
| 3D 뷰어(검증용) | 기존 Three.js/PLY 뷰어에 포신 축 오버레이 |
